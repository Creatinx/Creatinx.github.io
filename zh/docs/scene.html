Scene

实体及其组件的容器。管理实体生命周期、父子层级和组件查询。

#include <GryceRC/GryceScene.h>

class Scene {
public:
    Entity create_entity(const std::string& name);
    Entity create_entity(const std::string& name, Entity parent);
    void destroy_entity(Entity entity);
    Entity find_entity(const std::string& name) const;

    bool is_valid(Entity entity) const;
    size_t entity_count() const;

    template<typename... Components>
    QueryResult<Components...> query();

    template<typename... Components>
    QueryResult<Components...> query() const;

    void clear();
};

场景图

通过父指针维护实体层级。世界空间变换矩阵通过累加父变换计算。

auto root = scene.create_entity("Root");
auto child = scene.create_entity("Child", root);
auto grandchild = scene.create_entity("GrandChild", child);

// 世界变换 = root * child * grandchild 本地变换
auto& t = grandchild.get_component<Transform>();
Matrix4f world = t.world_matrix();

设计原理

Scene 是 gryce-engine 中的场景图容器，负责 entity 的生命周期管理和父子层级维护。它使用 ComponentStore 作为组件存储后端，通过 entity ID 索引所有组件。Scene 提供 create_entity 和 destroy_entity 操作，以及 find_entity 按名称查找。destroy_entity 递归销毁所有子实体，确保场景图的一致性。

查询系统

Scene 的 query 方法遍历所有拥有指定组件的实体，返回一个 QueryResult 对象，支持 range-based for 循环。query 内部使用 ComponentStore 的 entities_with 方法，保证 O(n) 时间复杂度，其中 n 是拥有该组件的实体数量。对于多组件查询，Scene 选择组件数量最少的类型作为遍历基础，然后过滤其他组件。

注意事项

  • create_entity 返回的 Entity 在 scene 中始终有效，直到 destroy_entity 被调用。
  • find_entity 按名称查找，如果有多个同名实体，返回第一个找到的。
  • clear 销毁所有实体和组件，但保留注册的类型信息。
  • query 在遍历过程中不能修改组件存储（添加/删除组件），否则会导致迭代器失效。
