实体组件系统

World

World 是 ECS 架构的中央协调器，持有 Scene 引用以及所有 ISystem 实例的列表。通过模板方法 add_system<T>() 注册系统，并自动按 priority() 降序排序，确保高优先级系统（如物理、输入）先于低优先级系统（如渲染）执行。

class World {
public:
    template<typename T, typename... Args>
    T* add_system(Args&&... args);

    void init();
    void fixed_update(float fixed_dt);  // 固定步长更新（物理）
    void update(float dt);              // 可变步长更新（逻辑）
    void render();                      // 生成渲染命令
    void cleanup();

private:
    Scene& scene;
    std::vector<std::unique_ptr<ISystem>> systems;
    // systems 按 priority() 降序排列
};

Entity

Entity 是轻量级句柄，内部仅存储实体 ID（EntityID）与指向 Scene 的指针。所有组件数据存储在 ComponentStore 中，Entity 本身不占额外内存。支持父子层级关系：子实体的 Transform 会相对于父实体进行局部变换。

class Entity {
public:
    static Entity create_entity(Scene& scene, const std::string& name = "");

    template<typename T, typename... Args>
    T& add_component(Args&&... args);

    template<typename T>
    T& get_component();

    template<typename T>
    bool has_component() const;

    template<typename T>
    void remove_component();

    void set_parent(Entity parent);
    Entity get_parent() const;
    const std::vector<EntityID>& get_children() const;
    void destroy();  // 标记销毁，延迟到帧末回收

private:
    EntityID id = INVALID_ENTITY;
    Scene* scene = nullptr;
};

ComponentStore

采用稀疏集（Sparse Set）存储的组件容器，兼顾 O(1) 随机访问与高速线性遍历。每个组件类型对应一个 ComponentStore<T> 实例，内部维护 sparse 数组（EntityID → 密集索引映射）与 dense 数组（实际组件数据）。

class IComponentStore {
public:
    virtual ~IComponentStore() = default;
    virtual void remove(EntityID entity) = 0;
};

template<typename T>
class ComponentStore : public IComponentStore {
public:
    void register_component();  // 在 World 中注册类型信息

    T& add(EntityID entity, const T& component);
    T& add(EntityID entity, T&& component);
    void remove(EntityID entity) override;
    T& get(EntityID entity);
    const T& get(EntityID entity) const;
    bool has(EntityID entity) const;

    // 线性遍历所有组件（Cache-friendly）
    std::vector<T>& data() { return dense; }
    const std::vector<EntityID>& entities() { return dense_entities; }

private:
    std::vector<size_t> sparse;      // 索引：EntityID，值：dense 索引
    std::vector<T> dense;            // 紧密排列的组件数据
    std::vector<EntityID> dense_entities;
};

ISystem

所有游戏系统的抽象基类。派生类需重写生命周期钩子，并可覆盖 priority() 以调整执行顺序（默认值为 0，数值越大优先级越高）。

class ISystem {
public:
    virtual ~ISystem() = default;
    virtual int priority() const { return 0; }

    virtual void init(World& world) {}
    virtual void fixed_update(World& world, float fixed_dt) {}
    virtual void update(World& world, float dt) {}
    virtual void render(World& world) {}
    virtual void cleanup(World& world) {}
};

完整使用示例

class PhysicsSystem : public ISystem {
public:
    int priority() const override { return 100; }  // 高优先级
    void fixed_update(World& world, float fixed_dt) override {
        // 遍历所有 Velocity + Transform 组件
    }
};

class RenderSystem : public ISystem {
public:
    int priority() const override { return 10; }   // 低优先级
    void render(World& world) override {
        // 提交 MeshRenderer 组件的绘制命令
    }
};

// 初始化
World world(scene);
world.add_system<PhysicsSystem>();
world.add_system<RenderSystem>();
world.init();

// 主循环
while (running) {
    world.fixed_update(1.0f / 60.0f);
    world.update(delta_time);
    world.render();
}
world.cleanup();

Gryce Engine

Gryce Engine 是一款从零手写的 C++23 游戏引擎。支持 OpenGL 4.6 与 Vulkan 1.2 双后端渲染，自定义 ECS，多线程命令缓冲区渲染，以及可插拔物理/音频模块。
