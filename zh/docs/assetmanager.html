AssetManager

AssetManager

已加载资源的中央注册表。支持热重载、引用计数和模板化类型加载。

#include <GryceRC/GryceAssetManager.h>

class AssetManager {
public:
    template<typename T>
    static Handle<T> load(const std::string& path);

    template<typename T>
    static Handle<T> get(const std::string& path);

    template<typename T>
    static bool is_loaded(const std::string& path);

    template<typename T>
    static void unload(const std::string& path);

    static void unload_all();
    static void reload_all();

    static void set_hot_reload(bool enabled);
    static bool hot_reload_enabled();

    static size_t memory_usage() const;
};

// 支持的资源类型
// Mesh, Texture, Shader, Material, AudioClip, Font, Scene

示例

auto mesh = AssetManager::load<Mesh>("res:/models/cube.obj");
auto texture = AssetManager::load<Texture>("res:/textures/diffuse.png");
auto shader = AssetManager::load<Shader>("res:/shaders/basic.glsl");

if (AssetManager::is_loaded<Mesh>("res:/models/cube.obj")) {
    auto same = AssetManager::get<Mesh>("res:/models/cube.obj");
}

设计原理

AssetManager 是 gryce-engine 的资源管理器，负责加载、缓存和卸载所有类型的资源。它使用静态模板方法实现类型安全的资源加载，通过 std::unordered_map<std::string, std::shared_ptr<T>> 缓存已加载资源。load() 在资源未加载时调用后端加载器，已加载时返回缓存的引用。这种设计保证同一资源不会被加载多次，减少内存占用和加载时间。AssetManager 支持热重载（hot reload），在资源文件修改后自动重新加载。

加载流程

load() 的调用流程：检查缓存 → 调用后端加载器 → 创建 GPU 资源 → 存入缓存 → 返回 Handle。对于 Mesh 和 Texture 等资源，后端加载器在后台线程读取文件，然后暂停渲染线程上传 GPU 数据。这种设计避免了主线程阻塞，但引入了 GPU 上传同步的复杂性。

注意事项

  • AssetManager 使用 shared_ptr 管理资源生命周期，只有所有 Handle 释放后资源才会卸载。
  • hot reload 通过定时检查文件修改时间实现，默认每 5 秒检查一次。
  • unload_all 会释放所有缓存，但不会影响正在使用的资源（shared_ptr 引用计数）。
  • memory_usage() 只统计 CPU 端内存，不包含 GPU 资源。
