VulkanDevice

VulkanDevice

选择具有图形和呈现队列的物理设备。创建逻辑设备和命令池。使用 VMA 进行内存管理。

#include <GryceRC/GryceVulkanDevice.h>

class VulkanDevice {
public:
    bool create(VkInstance instance, VkSurfaceKHR surface);
    void destroy();

    VkPhysicalDevice physical_device() const;
    VkDevice device() const;
    VmaAllocator allocator() const;

    uint32_t graphics_queue_family() const;
    uint32_t present_queue_family() const;
    VkQueue graphics_queue() const;
    VkQueue present_queue() const;

    VkCommandPool command_pool() const;
    VkCommandBuffer allocate_command_buffer() const;
    void free_command_buffer(VkCommandBuffer cmd) const;

    VkPhysicalDeviceProperties properties() const;
    bool supports_extension(const char* name) const;
};

设计原理

VulkanDevice 负责选择物理设备、创建逻辑设备和初始化 VMA（Vulkan Memory Allocator）。在 create() 中，它枚举所有可用的物理设备，选择第一个满足要求的设备（必须有图形和呈现队列）。然后创建逻辑设备，并初始化 VMA 分配器。VMA 提供了高效的 GPU 内存管理，自动处理 memory type 选择和 defragmentation。

队列族

Vulkan 的 queue family 是一个关键概念。graphics_queue_family 用于图形命令，present_queue_family 用于呈现到 surface。在大多数设备上这两个 queue family 是相同的，但在某些设备上可能不同。VulkanDevice 分别记录这两个 queue family 索引，并确保 command buffer 提交到正确的队列。

注意事项

  • create() 必须在 VulkanBackend 初始化之后调用，因为需要 instance 和 surface。
  • VMA 分配器在 destroy() 中销毁，必须在所有 GPU 资源释放之后。
  • command_pool 在创建时针对 graphics queue family 初始化。
  • allocate_command_buffer 创建的 command buffer 必须配对 free_command_buffer 或 submit 后由 Vulkan 回收。
