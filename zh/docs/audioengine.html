AudioEngine

AudioEngine

基于 miniaudio 的音频引擎。管理设备初始化、主音量和全局 listener 状态。

#include <GryceRC/GryceAudio.h>

class AudioEngine {
public:
    bool init();
    void shutdown();

    void set_master_volume(float volume); // 0.0 - 1.0
    float master_volume() const;

    void set_listener_position(const Vector3f& pos);
    void set_listener_direction(const Vector3f& forward, const Vector3f& up);

    AudioClipHandle load_clip(const std::string& path); // WAV, MP3, OGG, FLAC
    void unload_clip(AudioClipHandle handle);

    void play_one_shot(AudioClipHandle clip, float volume = 1.0f);
    void play_one_shot_3d(AudioClipHandle clip, const Vector3f& pos, float volume = 1.0f);

    size_t active_voice_count() const;
    void stop_all_voices();
};

示例

AudioEngine audio;
audio.init();

auto jump = audio.load_clip("res:/audio/jump.wav");
audio.set_master_volume(0.8f);

audio.play_one_shot(jump);

设计原理

AudioEngine 是 gryce-engine 的音频后端，基于 miniaudio 库。它管理音频设备的生命周期、主音量、全局 listener 位置和 3D 空间音频计算。AudioEngine 在 init() 中初始化 miniaudio 的 context 和 device，在 shutdown() 中释放所有资源。它支持 WAV、MP3、OGG 和 FLAC 格式，通过 miniaudio 的解码器自动识别格式。3D 音频使用 miniaudio 的 spatializer，计算声音源和 listener 之间的距离和方向衰减。

3D 音频

AudioEngine 通过 set_listener_position 和 set_listener_direction 设置全局 listener 状态。每个 AudioSource 组件在 update 时将自己的世界位置传递给 AudioEngine，AudioEngine 计算 attenuation 并应用到对应的声音源。miniaudio 的 spatializer 使用 HRTF（头部相关传递函数）模拟立体声定位，提供准确的 3D 音频效果。

注意事项

  • AudioEngine 的 init() 必须在任何音频操作之前调用。
  • 主音量范围是 0.0 - 1.0，超过此范围会被截断。
  • play_one_shot 创建临时声音源，播放结束后自动销毁。
  • active_voice_count() 返回当前播放的声音数量，可用于性能监控。
