Transform

核心组件，存储本地位置、旋转（欧拉角）和缩放。从父层级计算世界矩阵。

#include <GryceRC/GryceTransform.h>

struct Transform {
    Vector3f position;
    Vector3f rotation; // 欧拉角，单位为度
    Vector3f scale = {1,1,1};

    Matrix4f local_matrix() const;
    Matrix4f world_matrix() const;

    Vector3f forward() const;
    Vector3f right() const;
    Vector3f up() const;

    void look_at(const Vector3f& target, const Vector3f& up = {0,1,0});
    void translate(const Vector3f& delta);
    void rotate(const Vector3f& delta);
};

示例

auto entity = world.create_entity("Camera");
auto& transform = entity.add_component<Transform>();
transform.position = {0, 2, 5};
transform.look_at({0, 0, 0});

auto camera = entity.add_component<Camera>(60.0f, 16.0f/9.0f, 0.1f, 100.0f);
camera.view = transform.world_matrix().inverse();

设计原理

Transform 是 gryce-engine 中最基础的组件，存储实体的本地位置、旋转（欧拉角）和缩放。它通过 Scene 的父子层级自动计算世界空间变换矩阵。local_matrix() 将 position、rotation 和 scale 组合为列主序 Matrix4f。world_matrix() 递归累加父实体的变换，直到根实体。这种设计保证子实体的变换自动跟随父实体。

欧拉角与四元数

Transform 使用欧拉角（Vector3f，单位为度）表示旋转。欧拉角直观且易于编辑，但存在万向锁问题。对于需要平滑插值的场景，建议使用外部四元数库（如 glm）计算旋转后写入 Transform。Transform 的 look_at 方法使用一个简化版，适合相机等需要朝向目标的场景。

注意事项

  • rotation 是欧拉角，按 Y → X → Z 顺序应用（YXZ 顺序）。
  • world_matrix() 在每次调用时重新计算，不要在每帧中多次调用同一个 entity。
  • scale 的默认值是 (1,1,1)，设置为 (0,0,0) 会导致矩阵不可逆。
  • forward/right/up 方向基于当前 rotation 计算，不考虑父实体的旋转。
