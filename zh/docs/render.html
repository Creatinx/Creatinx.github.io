渲染系统

IRenderBackend

所有渲染后端的公共基类，声明了设备生命周期、帧循环、资源创建与绘制接口。应用层通过 IRenderBackend* 指针操作，无需关心底层是 OpenGL 还是 Vulkan。

class IRenderBackend {
public:
    virtual bool init(const RenderBackendDesc& desc) = 0;
    virtual void shutdown() = 0;
    virtual void begin_frame() = 0;
    virtual void end_frame() = 0;
    virtual void set_viewport(uint32_t x, uint32_t y, uint32_t w, uint32_t h) = 0;
    virtual void set_depth_test(bool enable) = 0;
    virtual void set_blend(bool enable, BlendFactor src, BlendFactor dst) = 0;
    virtual void clear(const ClearDesc& desc) = 0;
    virtual RenderResourceHandle create_mesh(const MeshDesc& desc) = 0;
    virtual RenderResourceHandle create_shader(const ShaderDesc& desc) = 0;
    virtual RenderResourceHandle create_texture(const TextureDesc& desc) = 0;
    virtual RenderResourceHandle create_framebuffer(const FramebufferDesc& desc) = 0;
    virtual void draw_mesh(RenderResourceHandle mesh, uint32_t instance_count = 1) = 0;
    virtual void draw_indexed(RenderResourceHandle mesh, uint32_t index_count, uint32_t index_offset = 0) = 0;
};

GLBackend

OpenGL 4.6 后端实现。初始化阶段调用 glewInit() 加载扩展，并在控制台输出 GPU vendor / renderer / version 日志。Windows 平台额外检测 WGL_NV_delay_before_swap 扩展，以支持更精细的帧同步控制。

GLShader

封装顶点与片段着色器的编译、链接及 uniform 上传。编译失败时自动打印 info log。所有 set_* 系列方法在 uniform 不存在时静默返回（get_uniform_location 返回 -1 时不触发错误）。

class GLShader {
public:
    bool compile(const std::string& vs_src, const std::string& fs_src);
    void bind() const;
    void set_int(const std::string& name, int value);
    void set_float(const std::string& name, float value);
    void set_vec2(const std::string& name, const Vector2f& v);
    void set_vec3(const std::string& name, const Vector3f& v);
    void set_vec4(const std::string& name, const Vector4f& v);
    void set_mat4(const std::string& name, const Matrix4f& m);
private:
    int get_uniform_location(const std::string& name) const;
    uint32_t program_id = 0;
};

GLMesh

管理 VAO / VBO / EBO 三元组。通过 upload_vertices() 将 CPU 顶点数据上传到 GPU；set_layout() 配置顶点属性指针；draw() 绑定 VAO 后执行 glDrawArrays 或 glDrawElements。

class GLMesh {
public:
    void upload_vertices(const void* data, size_t size, uint32_t vertex_count);
    void set_layout(const std::vector<VertexAttrib>& attribs);
    void set_index_data(const uint32_t* indices, uint32_t index_count);
    void draw() const;
    void draw_indexed() const;
private:
    uint32_t vao = 0, vbo = 0, ebo = 0;
    uint32_t vertex_count = 0, index_count = 0;
};

GLRenderer2D

CPU 端 batching 的 2D 渲染器。单帧内收集所有 draw_rect、draw_circle、draw_polygon、draw_text 调用，在 flush() 时一次性提交合并后的顶点数据，减少 draw call 数量。

class GLRenderer2D {
public:
    void draw_rect(const Vector2f& pos, const Vector2f& size, const Vector4f& color);
    void draw_circle(const Vector2f& center, float radius, const Vector4f& color, uint32_t segments = 32);
    void draw_polygon(const std::vector<Vector2f>& points, const Vector4f& color);
    void draw_text(const std::string& text, const Vector2f& pos, float scale, const Vector4f& color);
    void flush();  // 提交当前 batch 到 GPU
private:
    std::vector<Vertex2D> vertex_batch;
    std::shared_ptr<FontAtlas> font_atlas;
};

RenderCommandBuffer & RenderThread

多线程渲染的核心同步设施。RenderCommandBuffer 维护双缓冲数组与读写锁；RenderThread 在独立线程中循环等待并执行命令。

class RenderCommandBuffer {
public:
    void submit(const std::vector<RenderCommand>& cmds); // 主线程写入
    std::vector<RenderCommand> acquire();                // 渲染线程读取
    void swap();                                          // 交换双缓冲索引
private:
    std::array<std::vector<RenderCommand>, 2> buffers;
    std::mutex mtx;
    std::condition_variable cv;
    uint8_t write_index = 0;
    uint8_t read_index = 1;
    bool new_frame_ready = false;
};

class RenderThread {
public:
    void start(IRenderBackend* backend, RenderCommandBuffer* cmd_buffer);
    void stop();
    void thread_loop();
    void execute(const std::vector<RenderCommand>& cmds);
private:
    std::thread thread;
    std::atomic<bool> running{false};
    IRenderBackend* backend = nullptr;
    RenderCommandBuffer* cmd_buffer = nullptr;
};

Vulkan 后端

VulkanBackend

Vulkan 1.2 后端实现，负责 instance、surface、device、swapchain、command pools 与同步原语（fence / semaphore）的初始化与销毁。帧循环使用双缓冲或三缓冲的 command buffer 池。

VulkanDevice

封装物理设备枚举、队列族选择与逻辑设备创建。自动选择具备 VK_QUEUE_GRAPHICS_BIT 与 VK_QUEUE_COMPUTE_BIT 的独立队列族，并创建 graphics + present 双队列。集成 VMA（Vulkan Memory Allocator）用于所有 buffer / image 的内存分配。

VulkanSwapchain

管理 VkSwapchainKHR 生命周期，包括 surface 格式选择、present mode 选择（优先 VK_PRESENT_MODE_FIFO_KHR）、图像视图创建与帧缓冲区的动态重建（窗口大小变化时自动触发）。

VulkanBuffer / VulkanTexture

基于 VMA 分配的 GPU 资源。VulkanBuffer 支持 staging buffer 上传与 device-local 常驻模式。VulkanTexture 支持 2D / 2D array / cubemap，自动生成 mipmap chain 与合适的图像布局转换 barrier。

VulkanShader

从 SPIR-V 二进制模块创建 VkShaderModule，并基于反射信息自动构建 descriptor set layouts 与 pipeline layout。支持 uniform buffer、combined image sampler 与 storage buffer 三种 descriptor 类型。

class VulkanShader {
public:
    bool load_spirv(const std::vector<uint32_t>& vert_spv,
                    const std::vector<uint32_t>& frag_spv);
    void bind(VkCommandBuffer cmd);
    void push_constants(VkCommandBuffer cmd, const void* data, size_t size);
private:
    VkShaderModule vert_module = VK_NULL_HANDLE;
    VkShaderModule frag_module = VK_NULL_HANDLE;
    VkPipelineLayout pipeline_layout = VK_NULL_HANDLE;
    std::vector<VkDescriptorSetLayout> set_layouts;
};

Gryce Engine

Gryce Engine 是一款从零手写的 C++23 游戏引擎。支持 OpenGL 4.6 与 Vulkan 1.2 双后端渲染，自定义 ECS，多线程命令缓冲区渲染，以及可插拔物理/音频模块。
