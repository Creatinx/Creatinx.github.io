RenderThread

RenderThread

专用后台线程，循环等待命令并在活动后端上执行。使用 condition_variable 实现低延迟唤醒。

#include <GryceRC/GryceRenderThread.h>

class RenderThread {
public:
    void start(IRenderBackend* backend, RenderCommandBuffer* cmd_buf);
    void stop();
    bool is_running() const;

    void wait_for_frame(); // 阻塞直到当前帧完成
    void set_target_fps(int fps); // 0 = 无限制

    uint32_t frame_count() const;
    float last_frame_time_ms() const;

private:
    void thread_loop(); // 等待 condition_variable，执行获取的命令
};

示例

RenderThread rt;
RenderCommandBuffer cmd_buf;
GLBackend backend;
backend.init();

rt.start(&backend, &cmd_buf);

// 主循环
while (window.is_open()) {
    // ... 游戏逻辑 ...
    cmd_buf.submit({RenderCommandType::Present, {}});
    rt.wait_for_frame();
}

rt.stop();
backend.shutdown();

设计原理

RenderThread 是一个 std::thread 的包装器，专门用于在后台执行渲染命令。它在 start() 中创建线程并进入 thread_loop()，在 stop() 中设置退出标志并等待线程结束。thread_loop() 的核心逻辑是：acquire 命令 → 执行命令 → present → 等待下一帧。wait_for_frame() 允许主线程在提交 Present 命令后阻塞，直到渲染线程完成当前帧，从而实现帧同步。

帧率控制

set_target_fps() 允许限制渲染线程的最大帧率。当设置为非零值时，渲染线程在每帧结束时计算剩余的睡眠时间，并通过 std::this_thread::sleep_for() 进行补偿。这种简单的帧率控制适合大多数场景，但对于需要精确帧 pacing 的应用，建议使用平台特定的 API（如 NVIDIA 的 WGL_NV_delay_before_swap）。

注意事项

  • RenderThread 在 stop() 之前必须确保所有命令已执行完毕。
  • 不要从渲染线程调用 submit()，这会导致死锁。
  • frame_count() 和 last_frame_time_ms() 是统计信息，不保证实时精度。
  • target_fps 为 0 时渲染线程以最大速度运行，可能占用大量 CPU。
