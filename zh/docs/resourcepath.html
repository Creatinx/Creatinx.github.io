ResourcePath

ResourcePath

虚拟路径解析。res:/ 前缀解析至项目根目录。支持路径拼接和扩展名查询。

#include <GryceRC/GryceResourcePath.h>

class ResourcePath {
public:
    ResourcePath(const std::string& path);

    std::string resolved() const; // 绝对文件系统路径
    std::string virtual_path() const; // 原始 res:/path
    std::string extension() const;
    std::string filename() const;
    std::string directory() const;

    bool exists() const;
    size_t file_size() const;

    ResourcePath operator/(const std::string& subpath) const;

    static void set_root(const std::string& absolute_path);
    static std::string root();
};

// 示例
ResourcePath mesh("res:/models/cube.obj");
std::string abs = mesh.resolved(); // /home/user/project/models/cube.obj

ResourcePath textures("res:/textures");
auto diffuse = textures / "diffuse.png";

设计原理

ResourcePath 是 gryce-engine 的虚拟路径系统，将 res:/ 前缀映射到项目根目录。这种设计允许游戏代码使用统一的路径格式，而不关心实际文件系统的布局。ResourcePath 支持路径拼接（operator/）和常用查询（exists, extension, filename）。在构建时，res:/ 根目录通过 set_root() 设置，通常为 CMake 的 CMAKE_SOURCE_DIR。

路径解析

ResourcePath 的解析流程：1) 去除 res:/ 前缀；2) 与 root() 拼接为绝对路径；3) 规范化路径分隔符（统一为 /）。resolved() 返回的绝对路径可以直接传递给 std::ifstream 或 AssetManager 的加载器。exists() 使用 std::filesystem::exists 检查文件存在性，file_size() 使用 std::filesystem::file_size 获取大小。

注意事项

  • set_root 必须在任何路径解析之前调用，通常在 main() 开始时。
  • ResourcePath 不支持 ../ 路径遍历，出于安全考虑会被拒绝。
  • 虚拟路径必须以 res:/ 开头，否则会被当作普通文件路径处理。
  • operator/ 自动处理路径分隔符，不需要手动添加 /。
