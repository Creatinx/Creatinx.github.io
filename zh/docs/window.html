Window

基于 GLFW 的跨平台窗口抽象。处理输入事件、OpenGL/Vulkan 上下文创建与帧计时。

#include <GryceRC/GryceWindow.h>

class Window {
public:
    Window(const std::string& title, int width, int height);
    ~Window();

    bool is_open() const;
    void poll_events();
    void swap_buffers();
    float delta_time() const;

    int width() const;
    int height() const;

    void set_vsync(bool enabled);
    bool vsync() const;

    void set_title(const std::string& title);
    void set_size(int width, int height);

    void set_cursor_mode(CursorMode mode);
    bool is_key_pressed(KeyCode key) const;
    bool is_mouse_button_pressed(MouseButton btn) const;
    Vector2f mouse_position() const;

    using EventCallback = std::function;
    void set_event_callback(EventCallback callback);
};

事件系统

Window 通过单一回调传递事件。事件类型包括 WindowResize、WindowClose、KeyPress、KeyRelease、MouseMove、MouseButtonPress、MouseButtonRelease、MouseScroll。

struct Event {
    EventType type;
    union {
        WindowResizeData resize;
        KeyData key;
        MouseData mouse;
    };
};

window.set_event_callback([](const Event& e) {
    if (e.type == EventType::KeyPress && e.key.code == KeyCode::Escape) {
        window.close();
    }
});

设计原理

Window 基于 GLFW 3.3+ 构建，提供跨平台的窗口和输入抽象。它不直接暴露 GLFW 对象，而是通过 Event 结构体将所有事件（键盘、鼠标、窗口调整大小）统一传递给单一回调。这种设计允许游戏代码以统一方式处理输入，而不关心底层平台差异。Window 同时管理 OpenGL 上下文或 Vulkan 表面，与渲染后端解耦。

事件系统

所有输入事件通过 set_event_callback 注册的一个回调函数传递。Event 是一个 tagged union，包含事件类型和具体数据。回调在游戏逻辑的主线程上同步执行，因此可以在回调中直接修改游戏状态。常用事件类型包括：KeyPress、KeyRelease、MouseMove、MouseButtonPress、WindowResize、WindowClose。

注意事项

  • Window 必须在主线程创建，这是 GLFW 的限制。
  • poll_events() 必须在每帧开始时调用，否则事件会堆积。
  • delta_time() 在首次调用时返回 0，之后返回实际帧时间。
  • set_vsync(true) 会限制帧率为显示器刷新率，但可能增加输入延迟。
