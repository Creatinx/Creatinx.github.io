World

Scene 和 ISystem 实例列表的拥有容器。初始化前按 priority() 降序排序系统。驱动游戏循环。

#include <GryceRC/GryceECS.h>

class World {
public:
    World();
    ~World();

    void add_system(std::unique_ptr<ISystem> system);
    template<typename T> void add_system() { add_system(std::make_unique<T>()); }

    void init();
    void fixed_update(float dt);
    void update(float dt);
    void render(IRenderBackend* backend);
    void cleanup();

    Entity create_entity(const std::string& name);
    void destroy_entity(Entity entity);

    Scene& scene();
    const Scene& scene() const;

    template<typename T> T* get_system() const;
    std::vector<ISystem*> systems() const;
};

示例

World world;
world.add_system<PhysicsSystem>();
world.add_system<RenderSystem>();
world.init();

while (window.is_open()) {
    world.fixed_update(1.0f/60.0f);
    world.update(window.delta_time());
    world.render(backend);
}
world.cleanup();

设计原理

World 是 gryce-engine 中最顶层的游戏对象容器。它拥有一个 Scene（场景图）和一组 ISystem（系统）实例。World 的职责是驱动整个游戏循环：按固定时间步长调用 fixed_update，按帧时间调用 update，并在每帧结束时调用 render。系统按 priority() 返回值降序排序，优先级高的系统先执行。这种设计保证渲染系统在逻辑系统之后执行，物理系统优先于游戏逻辑。

生命周期

World 的生命周期严格分为五个阶段：init → fixed_update → update → render → cleanup。init 在程序启动时调用一次，cleanup 在退出时调用一次。fixed_update 保证以固定时间步长（通常 1/60 秒）调用，适合物理模拟。update 以实际帧时间调用，适合游戏逻辑。render 在每帧结束时调用，接收 IRenderBackend* 用于实际绘制。

注意事项

  • World 不是线程安全的，所有操作必须在主线程执行。
  • System 的 priority() 默认返回 0，值越大执行越早。
  • create_entity 返回的 Entity 句柄在对应 Entity 被销毁后失效。
  • scene() 返回的 Scene 引用在 World 生命周期内始终有效。
