GLBackend

GLBackend

OpenGL 4.6 后端实现。启动时执行 glewInit，记录 GPU 供应商信息，支持 NVIDIA WGL_NV_delay_before_swap 帧率控制。

#include <GryceRC/GryceGLBackend.h>

class GLBackend : public IRenderBackend {
public:
    bool init() override;
    void shutdown() override;

    void begin_frame() override;
    void end_frame() override;

    void set_viewport(int x, int y, int w, int h) override;
    void set_depth_test(bool enabled) override;
    void set_blend(bool enabled, BlendMode mode) override;
    void clear(float r, float g, float b, float a) override;

    MeshHandle create_mesh(const VertexLayout& layout) override;
    ShaderHandle create_shader(const std::string& vs, const std::string& fs) override;
    TextureHandle create_texture(int w, int h, const void* data) override;
    FramebufferHandle create_framebuffer(int w, int h) override;

    void draw_mesh(MeshHandle mesh, ShaderHandle shader) override;
    void draw_indexed(MeshHandle mesh, ShaderHandle shader, uint32_t index_count) override;

    void destroy_mesh(MeshHandle handle) override;
    void destroy_shader(ShaderHandle handle) override;
    void destroy_texture(TextureHandle handle) override;
    void destroy_framebuffer(FramebufferHandle handle) override;

    const char* vendor_string() const;
    const char* renderer_string() const;
    bool has_nv_delay_before_swap() const;
};

};

struct VertexLayout {
    size_t stride;
    std::vector<VertexAttribute> attributes;
};

设计原理

GLBackend 是 IRenderBackend 的 OpenGL 4.6 实现。它在 init() 中执行 glewInit() 并查询 GPU 能力，在 begin_frame() 和 end_frame() 中管理帧缓冲状态。GLBackend 直接使用 OpenGL API，没有额外的抽象层，因此性能开销最小。所有 GPU 资源（VAO、VBO、EBO、shader program、texture）在 create 时直接调用对应的 glGen* 函数。

性能特性

GLBackend 在 NVIDIA 硬件上支持 WGL_NV_delay_before_swap 扩展，用于精确帧率控制。它使用传统的 immediate mode 渲染（通过 draw_mesh 和 draw_indexed），适合快速原型开发和小型项目。对于高性能需求，建议使用 VulkanBackend。

注意事项

  • GLBackend 不支持多线程命令提交，所有 OpenGL 调用必须在渲染线程执行。
  • Shader 编译错误通过 spdlog 输出到控制台。
  • VAO 状态在 create_mesh 时绑定，draw 时不需要重新绑定。
  • 深度测试和混合状态在 set_depth_test / set_blend 中立即生效。
