VulkanBuffer

VMA 分配的 GPU 缓冲区，支持 staging 上传。可用于顶点、索引和 uniform 缓冲区。

#include <GryceRC/GryceVulkanBuffer.h>

class VulkanBuffer {
public:
    bool create(VulkanDevice* device, size_t size, VkBufferUsageFlags usage, VmaMemoryUsage mem_usage);
    void destroy();
    void upload(const void* data, size_t size);
    void upload_staged(const void* data, size_t size); // 使用 staging buffer

    VkBuffer handle() const;
    VkDeviceMemory memory() const;
    size_t size() const;
    void* mapped_ptr() const; // 不可主机访问时为 nullptr
};

class VulkanTexture {
public:
    bool create(VulkanDevice* device, int w, int h, const void* data);
    void destroy();

    VkImage image() const;
    VkImageView view() const;
    VkSampler sampler() const;
    VkDescriptorSet descriptor_set() const;

    int width() const;
    int height() const;
    VkFormat format() const;
};

设计原理

VulkanBuffer 和 VulkanTexture 是 gryce-engine 的 Vulkan GPU 资源管理类，使用 VMA（Vulkan Memory Allocator）进行内存分配。VMA 自动处理 memory type 选择、alignment 和 defragmentation，简化了 Vulkan 复杂的内存管理。VulkanBuffer 支持 staging upload：如果目标内存不可主机访问，upload_staged() 创建临时 staging buffer，使用 memcpy 填充数据，然后提交 copy command 到 GPU。

内存类型

Vulkan 的内存类型分为 HOST_VISIBLE（CPU 可访问）、DEVICE_LOCAL（GPU 本地）和 HOST_CACHED 等。VulkanBuffer 的 create 方法通过 VMA 的 memory usage hint（如 VMA_MEMORY_USAGE_GPU_ONLY）选择最优内存类型。对于频繁更新的数据（如 uniform buffer），建议使用 VMA_MEMORY_USAGE_CPU_TO_GPU，允许直接 CPU 写入。

注意事项

  • upload() 要求内存为 HOST_VISIBLE，否则必须使用 upload_staged()。
  • staging buffer 在 upload_staged 完成后自动销毁，不需要手动管理。
  • VulkanTexture 创建时自动生成 mipmap（如果支持），使用 VK_FORMAT_R8G8B8A8_UNORM。
  • 所有 VulkanBuffer 必须在 VulkanDevice 销毁之前释放，否则会导致 VMA 崩溃。
