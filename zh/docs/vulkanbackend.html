VulkanBackend

VulkanBackend

Vulkan 1.2 后端，管理 instance、surface、逻辑设备和 swapchain。使用 VMA 进行 GPU 内存分配。

#include <GryceRC/GryceVulkanBackend.h>

class VulkanBackend : public IRenderBackend {
public:
    bool init() override;
    void shutdown() override;

    void begin_frame() override;
    void end_frame() override;

    void set_viewport(int x, int y, int w, int h) override;
    void set_depth_test(bool enabled) override;
    void set_blend(bool enabled, BlendMode mode) override;
    void clear(float r, float g, float b, float a) override;

    MeshHandle create_mesh(const VertexLayout& layout) override;
    ShaderHandle create_shader(const std::string& vs, const std::string& fs) override;
    TextureHandle create_texture(int w, int h, const void* data) override;
    FramebufferHandle create_framebuffer(int w, int h) override;

    void draw_mesh(MeshHandle mesh, ShaderHandle shader) override;
    void draw_indexed(MeshHandle mesh, ShaderHandle shader, uint32_t index_count) override;

    void destroy_mesh(MeshHandle handle) override;
    void destroy_shader(ShaderHandle handle) override;
    void destroy_texture(TextureHandle handle) override;
    void destroy_framebuffer(FramebufferHandle handle) override;

    VkInstance instance() const;
    VkSurfaceKHR surface() const;
    VulkanDevice* device() const;
    VulkanSwapchain* swapchain() const;
};

设计原理

VulkanBackend 是 IRenderBackend 的 Vulkan 1.2 实现。它管理 Vulkan instance、surface、device、swapchain 和 command buffer 的完整生命周期。与 GLBackend 不同，VulkanBackend 使用显式的同步原语（semaphore、fence）来管理 CPU-GPU 并行。所有资源创建和销毁操作都被封装在对应的管理类（VulkanDevice、VulkanSwapchain、VulkanBuffer 等）中，避免在 VulkanBackend 主类中积累过多的平台特定代码。

多线程注意事项

Vulkan 允许在多个线程上构建 command buffer，但 gryce-engine 当前只在渲染线程上使用单个 command buffer。这种简化保证了正确性，但限制了 Vulkan 的多线程构建优势。未来的版本可能支持 secondary command buffer 和并行构建。

注意事项

  • VulkanBackend 要求 Vulkan 1.2 兼容的驱动和硬件。
  • surface 创建需要平台特定的扩展（Windows 上为 VK_KHR_win32_surface）。
  • swapchain 在窗口大小改变时必须调用 recreate()。
  • Vulkan 的 validation layers 在 Debug 构建中启用，Release 构建中禁用。
