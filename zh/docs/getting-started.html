快速开始

环境要求

Gryce Engine 在 Windows 平台采用 MSYS2 UCRT64 工具链进行开发。开始之前，请确保系统满足以下最低要求：

  • 操作系统：Windows 10/11（64-bit）
  • 编译环境：MSYS2 UCRT64
  • 编译器：GCC 13+（需完整支持 C++23）
  • 构建工具：CMake 3.28+、Ninja
  • 脚本依赖：Python 3.10+（用于 build.py 一键构建）

安装步骤

1. 克隆仓库到本地：

git clone https://github.com/Creatinx/Gryce-Engine.git
cd Gryce-Engine

2. 在 MSYS2 UCRT64 终端中安装系统级依赖：

pacman -S mingw-w64-ucrt-x86_64-gcc \
          mingw-w64-ucrt-x86_64-cmake \
          mingw-w64-ucrt-x86_64-ninja \
          mingw-w64-ucrt-x86_64-glew \
          mingw-w64-ucrt-x86_64-glfw

3. 使用 build.py 执行首次编译（默认 Debug 配置）：

python build.py --jobs 8

4. 运行生成的可执行文件进行验证：

./build/Debug/bin/Debug/gryce.exe

项目目录结构

仓库根目录采用清晰的模块化布局：

Gryce-Engine/
├── core/          # 引擎核心库（渲染、ECS、物理、音频等）
├── editor/        # 编辑器可执行文件
├── tests/         # 单元测试（基于 GTest）
├── examples/      # 示例项目与 Demo
├── cmake/         # CMake 自定义模块与工具链文件
├── docs/          # 文档与网站源文件
├── third_party/   # 外部依赖缓存（FetchContent / 手动放置）
└── build.py       # 一键构建脚本（Debug/Release/Clean）

build.py 使用示例

build.py 是项目的入口构建脚本，封装了 CMake 配置、依赖获取、编译与清理的完整流程：

# Debug 配置并行构建（默认）
python build.py --jobs 8

# Release 配置，启用 LTO 与全量优化
python build.py Release --jobs 8

# 仅清理构建产物，保留第三方依赖缓存
python build.py --clean

# 完全清理（包括重新下载外部依赖）
python build.py --clean-all

# 输出完整编译命令，用于排查构建错误
python build.py --verbose
命令说明
--jobs N指定并行编译任务数，建议与 CPU 逻辑核心数一致
--clean删除 build/ 目录但保留 third_party/ 缓存
--clean-all彻底清理 build/ 与 third_party/，下次构建将重新拉取依赖
--verbose打印完整 ninja 编译命令与 CMake 配置日志

G
Gryce Engine

Gryce Engine 是一款从零手写的 C++23 游戏引擎。支持 OpenGL 4.6 与 Vulkan 1.2 双后端渲染，自定义 ECS，多线程命令缓冲区渲染，以及可插拔物理/音频模块。
