IRenderBackend

IRenderBackend

RHI（渲染硬件接口）核心抽象。由 OpenGL 和 Vulkan 后端分别实现。游戏代码不直接调用后端方法，而是向 RenderCommandBuffer 提交命令。

#include <GryceRC/GryceRender.h>

class IRenderBackend {
public:
    virtual ~IRenderBackend() = default;

    virtual bool init() = 0;
    virtual void shutdown() = 0;

    virtual void begin_frame() = 0;
    virtual void end_frame() = 0;

    virtual void set_viewport(int x, int y, int w, int h) = 0;
    virtual void set_depth_test(bool enabled) = 0;
    virtual void set_blend(bool enabled, BlendMode mode) = 0;
    virtual void clear(float r, float g, float b, float a) = 0;

    virtual MeshHandle create_mesh(const VertexLayout& layout) = 0;
    virtual ShaderHandle create_shader(const std::string& vs, const std::string& fs) = 0;
    virtual TextureHandle create_texture(int w, int h, const void* data) = 0;
    virtual FramebufferHandle create_framebuffer(int w, int h) = 0;

    virtual void draw_mesh(MeshHandle mesh, ShaderHandle shader) = 0;
    virtual void draw_indexed(MeshHandle mesh, ShaderHandle shader, uint32_t index_count) = 0;

    virtual void destroy_mesh(MeshHandle handle) = 0;
    virtual void destroy_shader(ShaderHandle handle) = 0;
    virtual void destroy_texture(TextureHandle handle) = 0;
    virtual void destroy_framebuffer(FramebufferHandle handle) = 0;
};

设计原理

IRenderBackend 是 gryce-engine 渲染架构的核心抽象。它定义了所有图形 API（OpenGL 4.6、Vulkan 1.2）必须实现的统一接口。游戏代码从不直接调用后端方法，而是通过 RenderCommandBuffer 提交命令。这种设计允许主线程生成渲染命令，而渲染线程在后台执行，实现 CPU/GPU 并行。IRenderBackend 的生命周期由 RenderContext 管理，确保正确的初始化顺序和资源释放。

资源管理

所有图形资源（Mesh、Shader、Texture、Framebuffer）通过 Handle 类型引用。Handle 是一个轻量级的整数 ID，背后由后端管理实际的 GPU 资源。create_* 方法返回 Handle，destroy_* 方法释放对应资源。后端负责资源的生命周期管理和内存回收。如果试图使用已销毁的 Handle，后端行为未定义（通常会导致渲染错误）。

注意事项

  • IRenderBackend 必须在渲染线程上执行命令。
  • 不要在主线程直接调用 draw 方法，使用 RenderCommandBuffer。
  • create_* 和 destroy_* 调用必须成对出现，防止内存泄漏。
  • Vulkan 后端需要额外的同步（semaphore/fence），由后端内部处理。
