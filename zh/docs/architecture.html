渲染线程 + 命令缓冲区

引擎使用多线程渲染架构：

  • 主线程：生成渲染命令，推入 RenderCommandBuffer
  • 渲染线程：从缓冲区获取命令，通过 IRenderBackend 执行
  • 同步：双缓冲队列，使用 std::mutex + std::condition_variable

RHI 抽象层

IRenderBackend 接口为 OpenGL 和 Vulkan 提供统一 API：

  • 生命周期：init() / shutdown()
  • 帧循环：begin_frame() / end_frame()
  • 状态：set_viewport、set_depth_test、set_blend、clear
  • 资源：create_mesh、create_shader、create_texture
  • 绘制：draw_mesh、draw_indexed

Gryce Engine

Gryce Engine 是一个从零手写的 C++ 游戏引擎，主要用于学习图形 API、编译器周边工具与引擎架构。没有使用现成引擎框架。
