Gryce-Engine

一个免费的、开源的 C++23 游戏引擎框架。

手写 ECS、RHI 与数学库。OpenGL 4.6 / Vulkan 1.2 双后端渲染。

快速开始 GitHub

核心特性

Gryce-Engine 目前处于早期开发阶段。

多后端渲染

OpenGL 4.6 与 Vulkan 1.2 双后端，统一 RHI 抽象层。渲染线程 + 命令缓冲区架构，实现 CPU/GPU 并行工作负载。

  • 双缓冲命令队列，mutex + condition_variable 同步
  • 类型化渲染命令，支持状态合并批处理优化
  • NVIDIA WGL_NV_delay_before_swap 帧率控制

实体组件系统

基于稀疏集的 ECS，自动组件池化。Godot/Unity 混合风格实体 API，支持父子层级关系。

  • 稀疏集存储，O(1) 组件查找
  • 每类型自动组件池化
  • 生命周期阶段：init、fixed_update、update、render、cleanup

物理集成

可插拔物理后端：Jolt Physics 3D、Box2D 2D，以及轻量级内置 2D 物理世界。

  • Jolt Physics 5.2：刚体动力学、碰撞检测、射线检测
  • Box2D 3.0：刚体、fixture、关节
  • 内置 2D：零外部依赖，显式 opt-in

空间音频

基于 miniaudio 的音频引擎，支持 3D 空间音频。AudioSource 与 AudioListener 组件可附加到实体。

  • 3D 位置音频与 listener 朝向
  • 每 AudioSource 独立控制循环、音量、播放/停止
  • 无外部音频服务器依赖

资源管线

后台线程异步资源加载。支持 OBJ、FBX、glTF（通过 Assimp）、PNG/JPG 纹理，以及自定义 .gesc 场景格式。

  • 后台线程异步加载器，支持回调
  • res:/ 虚拟路径解析至项目根目录
  • GPU 安全上传，支持暂停渲染线程

自定义数学库

列主序 Matrix4f，支持 ortho、perspective、look_at、rotate、scale、translate。Vector2f/3f/4f 完整运算符重载。

  • 列主序 Matrix4f：m[col * 4 + row]
  • 向量完整运算符重载
  • 无外部数学库依赖（glm 可选）

多线程渲染

主线程生成命令，渲染线程执行。Mutex + condition_variable 同步，双缓冲机制。

RHI 抽象

IRenderBackend 接口提供统一 API。运行时切换 OpenGL 和 Vulkan，无需修改游戏代码。

类型安全命令

RenderCommandTyped 结构体配合 RenderCommandType 枚举。连续状态设置命令可合并为批处理优化。

代码示例

editor/main.cpp 
// gryce-engine — Minimal setup example
#include <GryceRC/GryceEngine.h>

using namespace gryce_engine;

int main() {
    Window window("Gryce-Engine Demo", 1280, 720);
    auto context = RenderContext::create(RenderBackend::OpenGL, window);
    World world;
    auto camera = world.scene().create_entity("Camera");
    camera.add_component<Camera>(60.0f, 16.0f/9.0f, 0.1f, 1000.0f);
    auto mesh = AssetManager::load<Mesh>("res:/models/cube.obj");
    auto shader = AssetManager::load<Shader>("res:/shaders/basic.glsl");
    while (window.is_open()) {
        window.poll_events();
        world.update(window.delta_time());
        world.render();
        context->present();
    }
    return 0;
}

快速开始

克隆、构建、运行

1

克隆仓库

从 GitHub 获取源代码。仓库包含核心引擎、编辑器、测试和示例。

git clone https://github.com/Creatinx/Gryce-Engine.git && cd Gryce-Engine
2

安装依赖（MSYS2 UCRT64）

Gryce-Engine 使用 GCC (C++23) 在 Windows 上通过 MSYS2 UCRT64 构建。安装编译器、构建工具和原生依赖。

pacman -S mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja mingw-w64-ucrt-x86_64-glew mingw-w64-ucrt-x86_64-glfw

可选：mingw-w64-ucrt-x86_64-assimp, mingw-w64-ucrt-x86_64-gtest

3

一键构建

python build.py 自动检测 GCC，运行 CMake + Ninja，并预取可选依赖。--jobs 启用并行编译。

python build.py --jobs 8

Release 模式：python build.py Release --jobs 8

4

运行编辑器

默认构建输出 3D 立方体演示 + 2D 叠加层。交互模式下窗口持续运行，CI 模式下约 15 秒后自动退出。

./build/Debug/bin/Debug/gryce.exe

MinGW DLL（libgcc、libstdc++、glew32、glfw3）自动复制到输出目录。
