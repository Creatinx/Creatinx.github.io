Gryce-Engine

A free, open-source C++23 game engine framework.

Handwritten ECS, RHI, and math library. Dual-backend rendering: OpenGL 4.6 / Vulkan 1.2.

Get Started GitHub

Core Features

Gryce-Engine is currently in early development.

Multi-Backend Rendering

OpenGL 4.6 and Vulkan 1.2 backends with unified RHI abstraction. Render thread + command buffer architecture for parallel CPU/GPU workloads.

  • Double-buffered command queue with mutex + condition_variable
  • Typed render commands with batch merging optimization
  • WGL_NV_delay_before_swap frame pacing on NVIDIA

Entity Component System

Sparse-set based ECS with automatic component pooling. Godot/Unity hybrid Entity API with parent-child hierarchy support.

  • O(1) component lookup via sparse-set storage
  • Automatic component pooling per type
  • Lifecycle phases: init, fixed_update, update, render, cleanup

Physics Integration

Pluggable physics backends: Jolt Physics 3D, Box2D 2D, and a lightweight built-in 2D physics world.

  • Jolt Physics 5.2: rigid body, collision, raycast
  • Box2D 3.0: bodies, fixtures, joints
  • BuiltIn 2D: zero-dependency, opt-in only

Spatial Audio

miniaudio-powered audio engine with 3D spatial audio. AudioSource and AudioListener components attach to Entities.

  • 3D positional audio with listener orientation
  • Loop, volume, play/stop per AudioSource
  • Zero external audio server dependency

Asset Pipeline

Async asset loading with background threads. Supports OBJ, FBX, glTF via Assimp, PNG/JPG textures, and custom .gesc scene formats.

  • Background-thread async loader with callbacks
  • res:/ virtual path resolves to project root
  • GPU-safe upload with render thread pause

Custom Math Library

Column-major Matrix4f with ortho, perspective, look_at, rotate, scale, translate. Vector2f/3f/4f with full operator overloading.

  • Column-major Matrix4f: m[col * 4 + row]
  • Full operator overloading for vectors
  • No external math dependency (glm is optional)

Multi-threaded Rendering

Main thread generates commands while render thread executes them. Mutex + condition_variable synchronization with double buffering.

RHI Abstraction

IRenderBackend interface provides a unified API. Swap between OpenGL and Vulkan at runtime without changing game code.

Type-safe Commands

RenderCommandTyped structs with RenderCommandType enum. Consecutive state-setting commands are merged as a batch optimization.

Example Code

editor/main.cpp 
// gryce-engine — Minimal setup example
#include <GryceRC/GryceEngine.h>

using namespace gryce_engine;

int main() {
    Window window("Gryce-Engine Demo", 1280, 720);
    auto context = RenderContext::create(RenderBackend::OpenGL, window);
    World world;
    auto camera = world.scene().create_entity("Camera");
    camera.add_component<Camera>(60.0f, 16.0f/9.0f, 0.1f, 1000.0f);
    auto mesh = AssetManager::load<Mesh>("res:/models/cube.obj");
    auto shader = AssetManager::load<Shader>("res:/shaders/basic.glsl");
    while (window.is_open()) {
        window.poll_events();
        world.update(window.delta_time());
        world.render();
        context->present();
    }
    return 0;
}

Quick Start

Clone, build, and run

1

Clone the repository

Get the source code from GitHub. The repo includes core engine, editor, tests, and examples.

git clone https://github.com/Creatinx/Gryce-Engine.git && cd Gryce-Engine
2

Install dependencies (MSYS2 UCRT64)

Gryce-Engine is built with GCC (C++23) on Windows via MSYS2 UCRT64. Install compiler, build tools, and native dependencies.

pacman -S mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja mingw-w64-ucrt-x86_64-glew mingw-w64-ucrt-x86_64-glfw

Optional: mingw-w64-ucrt-x86_64-assimp, mingw-w64-ucrt-x86_64-gtest

3

Build with one command

The python build.py script auto-detects GCC, runs CMake + Ninja, and prefetches optional dependencies. Add --jobs for parallel compilation.

python build.py --jobs 8

For release: python build.py Release --jobs 8

4

Run the editor

The default build produces a 3D cube demo with 2D overlay. It runs for ~15 seconds in CI mode, or stays open in interactive mode.

./build/Debug/bin/Debug/gryce.exe

MinGW DLLs (libgcc, libstdc++, glew32, glfw3) are auto-copied to output directory.
