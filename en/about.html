About Gryce

An early-stage C++23 game framework by Creatinx. No editor yet.

Gryce is an early-stage C++23 game framework by Creatinx. It is not a full engine — there is no editor, no asset pipeline GUI, and no visual scripting. What exists is a collection of handwritten core layers (ECS, RHI, math, scene graph) built on top of mature third-party libraries.

Third-party dependencies: GLFW (window/input), GLEW (OpenGL loader), Vulkan SDK + VMA (Vulkan backend), miniaudio (audio), Jolt Physics (3D physics), Box2D (2D physics), Assimp (model loading), stb_image (textures), nlohmann/json (scenes), glm (optional math), spdlog (logging). These are not optional — the framework will not compile without them.

Handwritten layers: Entity Component System (sparse-set storage), Render Hardware Interface (OpenGL + Vulkan backends), custom math library (Vector2f/3f/4f, Matrix4f), scene graph (Entity, Transform, prefab), async asset loader, render thread + command buffer architecture.

Developed on Windows with MSYS2 UCRT64 GCC (C++23), CMake + Ninja. Python build.py for one-click builds.
