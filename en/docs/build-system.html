Build System

build.py

One-click build script that auto-detects GCC, runs CMake + Ninja, and prefetches optional dependencies.

# Debug build (default)
python build.py --jobs 8

# Release build
python build.py Release --jobs 8

# Clean and rebuild
python build.py --clean --jobs 8

# With tests
python build.py --tests --jobs 8

CMake Options

OptionDefaultDescription
Gryce_BUILD_EDITORONBuild editor executable
Gryce_BUILD_TESTSOFFBuild unit tests
Gryce_BACKEND_OPENGLONEnable OpenGL 4.6 backend
Gryce_BACKEND_VULKANONEnable Vulkan 1.2 backend
Gryce_USE_JOLTONEnable Jolt Physics 3D
Gryce_USE_BOX2DONEnable Box2D 2D physics
Gryce_USE_ASSIMPONEnable Assimp model loading

Manual CMake Build

cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release   -DCMAKE_CXX_COMPILER=g++ -DCMAKE_C_COMPILER=gcc
ninja -C build -j8

设计原理

gryce-engine 的构建系统基于 CMake 3.28+，使用预设（preset）和工具链文件（toolchain file）来自动检测 MSYS2 UCRT64 环境。build.py 是一个 Python 包装脚本，解析命令行参数，选择正确的 preset（debug 或 release），然后调用 cmake --build。它还支持自动安装缺失的依赖（通过 pacman 包管理器）和预取可选的第三方库（如 Jolt、Box2D）。

CMake 选项

Gryce_BUILD_EDITOR 控制是否构建编辑器可执行文件。Gryce_BUILD_TESTS 控制是否编译 GoogleTest 单元测试。Gryce_BACKEND_OPENGL 和 Gryce_BACKEND_VULKAN 分别控制 OpenGL 和 Vulkan 后端的编译。Gryce_USE_JOLT、Gryce_USE_BOX2D 和 Gryce_USE_ASSIMP 控制可选物理和模型加载库。所有选项默认为 ON（除了 TESTS），如果对应库未安装则自动降级为 OFF。

注意事项

  • build.py 假设 MSYS2 安装在默认路径 C:\msys64，如果安装在其他位置需要手动设置 PATH。
  • CMake 使用 Ninja 生成器，比 Make 快 2-3 倍，但首次运行需要安装 ninja。
  • Release 构建启用 -O3 和 LTO，但编译时间比 Debug 长 3-5 倍。
  • --clean 标志会删除整个 build 目录，谨慎使用。
G
gryce-engine

A very early-stage C++23 game framework. No editor. Not built from scratch — depends on GLFW, GLEW, Vulkan SDK, miniaudio, Jolt, Box2D, Assimp, and others. Handwritten parts: ECS, RHI, math, scene.
