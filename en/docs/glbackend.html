GLBackend

GLBackend

OpenGL 4.6 backend implementation. Performs glewInit on startup, logs GPU vendor info, and supports WGL_NV_delay_before_swap for frame pacing on NVIDIA hardware.

#include <GryceRC/GryceGLBackend.h>

class GLBackend : public IRenderBackend {
public:
    bool init() override;
    void shutdown() override;

    void begin_frame() override;
    void end_frame() override;

    void set_viewport(int x, int y, int w, int h) override;
    void set_depth_test(bool enabled) override;
    void set_blend(bool enabled, BlendMode mode) override;
    void clear(float r, float g, float b, float a) override;

    MeshHandle create_mesh(const VertexLayout& layout) override;
    ShaderHandle create_shader(const std::string& vs, const std::string& fs) override;
    TextureHandle create_texture(int w, int h, const void* data) override;
    FramebufferHandle create_framebuffer(int w, int h) override;

    void draw_mesh(MeshHandle mesh, ShaderHandle shader) override;
    void draw_indexed(MeshHandle mesh, ShaderHandle shader, uint32_t index_count) override;

    void destroy_mesh(MeshHandle handle) override;
    void destroy_shader(ShaderHandle handle) override;
    void destroy_texture(TextureHandle handle) override;
    void destroy_framebuffer(FramebufferHandle handle) override;

    const char* vendor_string() const;
    const char* renderer_string() const;
    bool has_nv_delay_before_swap() const;
};

VertexLayout

Describes vertex attribute format for mesh creation.

struct VertexAttribute {
    std::string name;
    uint32_t location;
    uint32_t components; // 1, 2, 3, or 4
    AttributeType type;  // float, int, uint
    bool normalized;
    size_t offset;
};

struct VertexLayout {
    size_t stride;
    std::vector<VertexAttribute> attributes;
};

Design Principles

GLBackend is the OpenGL 4.6 implementation of IRenderBackend. It executes glewInit() in init() and queries GPU capabilities, managing framebuffer state in begin_frame() and end_frame(). GLBackend directly uses OpenGL APIs without an additional abstraction layer, minimizing performance overhead. All GPU resources (VAO, VBO, EBO, shader program, texture) call the corresponding glGen* functions at creation time.

Performance Characteristics

GLBackend supports the WGL_NV_delay_before_swap extension on NVIDIA hardware for precise frame rate control. It uses traditional immediate mode rendering (via draw_mesh and draw_indexed), suitable for rapid prototyping and small projects. For high-performance needs, VulkanBackend is recommended.

Notes

  • GLBackend does not support multi-threaded command submission; all OpenGL calls must be executed on the render thread.
  • Shader compilation errors are output to the console via spdlog.
  • VAO state is bound at create_mesh time; no re-binding is needed during draw.
  • Depth testing and blending states take effect immediately in set_depth_test / set_blend.

gryce-engine

A very early-stage C++23 game framework. No editor. Not built from scratch — depends on GLFW, GLEW, Vulkan SDK, miniaudio, Jolt, Box2D, Assimp, and others. Handwritten parts: ECS, RHI, math, scene.
