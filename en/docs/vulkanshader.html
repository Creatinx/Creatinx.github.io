VulkanShader

VulkanShader

SPIR-V modules with descriptor set layouts and pipeline layouts. Supports push constants for small per-draw data.

#include <GryceRC/GryceVulkanShader.h>

class VulkanShader {
public:
    bool load_spirv(const std::string& vert_path, const std::string& frag_path);
    bool load_glsl(const std::string& vert_src, const std::string& frag_src); // auto-compiles to SPIR-V
    void destroy();

    void bind(VkCommandBuffer cmd);
    void set_push_constant(VkCommandBuffer cmd, const std::string& name, const void* data);

    VkPipelineLayout pipeline_layout() const;
    VkDescriptorSetLayout descriptor_set_layout() const;
    VkPipeline graphics_pipeline(VkRenderPass render_pass, const VertexLayout& layout) const;

    uint32_t push_constant_size() const;
    uint32_t push_constant_offset(const std::string& name) const;
};

Example

VulkanShader shader;
shader.load_spirv("res:/shaders/basic.vert.spv", "res:/shaders/basic.frag.spv");

Matrix4f mvp = projection * view * model;
shader.bind(cmd);
shader.set_push_constant(cmd, "u_mvp", mvp.data());

设计原理

VulkanShader 管理 Vulkan 的 shader module、pipeline layout、descriptor set layout 和 push constant 范围。它支持加载预编译的 SPIR-V 文件（.spv）和从 GLSL 源码动态编译（通过 glslangValidator）。pipeline layout 在 load_spirv 时创建，包含所有 uniform 和 push constant 的绑定信息。VulkanShader 使用 push constant 传递 MVP 矩阵等小型数据，避免了 descriptor set 更新的开销。

Push Constant

Push constant 是 Vulkan 的一种高效数据传递机制，允许在 command buffer 录制时直接嵌入最多 128 字节的数据。VulkanShader 的 set_push_constant 方法将数据复制到 command buffer 中，无需内存分配或同步。这种设计适合每帧变化的 uniform（如 model-view-projection 矩阵），但对于大量数据（如材质参数）仍需要使用 descriptor set。

注意事项

  • SPIR-V 文件必须在编译时通过 glslangValidator 生成，运行时不支持 GLSL 编译。
  • push constant 大小在 Vulkan 规范中限制为 128 字节，超出此限制会导致验证错误。
  • pipeline_layout() 在 load_spirv 之后有效，销毁 shader 后失效。
  • graphics_pipeline() 根据当前 render pass 和 vertex layout 创建 pipeline，缓存已创建的 pipeline 避免重复。
