RenderCommandBuffer

RenderCommandBuffer

Double-buffered command queue with mutex and condition variable. Main thread submit()s commands; render thread acquire()s them. Consecutive state-setting commands are merged automatically.

#include <GryceRC/GryceRenderCommand.h>

enum class RenderCommandType {
    Clear, SetViewport, SetDepthTest, SetBlend,
    DrawMesh, DrawIndexed, DrawMeshInstanced,
    SetShader, SetUniform, SetTexture,
    BeginFrame, EndFrame, Present
};

struct RenderCommand {
    RenderCommandType type;
    union {
        ClearData clear;
        ViewportData viewport;
        DrawData draw;
        UniformData uniform;
    };
};

class RenderCommandBuffer {
public:
    void submit(RenderCommand cmd);
    void submit(std::vector<RenderCommand>& commands);
    std::vector<RenderCommand> acquire(); // blocks until available

    void swap_buffers();
    bool is_empty() const;
    size_t pending_count() const;

    void set_merge_batch(bool enabled); // default true
};

Example

RenderCommandBuffer cmd_buf;

cmd_buf.submit({RenderCommandType::Clear, {.clear={0.1f,0.1f,0.1f,1.0f}}});
cmd_buf.submit({RenderCommandType::SetViewport, {.viewport={0,0,1280,720}}});
cmd_buf.submit({RenderCommandType::DrawMesh, {.draw={mesh_handle, shader_handle}}});

auto cmds = cmd_buf.acquire();
for (auto& c : cmds) backend->execute(c);

设计原理

RenderCommandBuffer 实现了一个双缓冲命令队列，使用 std::mutex 和 std::condition_variable 进行线程同步。主线程通过 submit() 将 RenderCommand 追加到写入缓冲区，渲染线程通过 acquire() 获取读取缓冲区中的命令并执行。当 acquire() 发现读取缓冲区为空时，它会等待 condition_variable 直到主线程调用 swap_buffers() 交换两个缓冲区。这种设计消除了主线程和渲染线程之间的竞争条件。

批处理合并

RenderCommandBuffer 可选地启用批处理合并（set_merge_batch(true)）。当连续提交多个状态设置命令（如 SetViewport、SetDepthTest）时，合并机制会将它们合并为单个命令，减少渲染线程的命令处理开销。批处理合并默认启用，在命令类型频繁切换的场景下可显著提升性能。

注意事项

  • submit() 是线程安全的，可以在任意线程调用。
  • acquire() 在读取缓冲区为空时会阻塞，直到 swap_buffers() 被调用。
  • RenderCommand 的 union 成员大小有限，大数据（如矩阵）应通过指针传递。
  • swap_buffers() 必须在主线程调用，通常在每帧结束时。

A very early-stage C++23 game framework. No editor. Not built from scratch — depends on GLFW, GLEW, Vulkan SDK, miniaudio, Jolt, Box2D, Assimp, and others. Handwritten parts: ECS, RHI, math, scene.
