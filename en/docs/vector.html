Vector

Vector2f / Vector3f / Vector4f

Handwritten vector types with full operator overloading. No external dependency (glm is optional).

#include <GryceRC/GryceMath.h>

struct Vector2f {
    float x, y;
    Vector2f() = default;
    Vector2f(float x, float y);

    float length() const;
    float length_sq() const;
    Vector2f normalized() const;
    void normalize();
    float dot(const Vector2f& other) const;
    float cross(const Vector2f& other) const;

    Vector2f operator+(const Vector2f& other) const;
    Vector2f operator-(const Vector2f& other) const;
    Vector2f operator*(float scalar) const;
    Vector2f operator/(float scalar) const;
    Vector2f& operator+=(const Vector2f& other);
    Vector2f& operator-=(const Vector2f& other);
    Vector2f& operator*=(float scalar);
    Vector2f& operator/=(float scalar);
    bool operator==(const Vector2f& other) const;
    bool operator!=(const Vector2f& other) const;
    float operator[](int index) const;
    float& operator[](int index);
};

// Vector3f adds cross product, reflect, lerp, slerp
struct Vector3f {
    float x, y, z;
    // ... same interface as Vector2f plus:
    Vector3f cross(const Vector3f& other) const;
    Vector3f reflect(const Vector3f& normal) const;
    Vector3f lerp(const Vector3f& target, float t) const;
    Vector3f slerp(const Vector3f& target, float t) const;
};

// Vector4f adds w component, homogenous division
struct Vector4f {
    float x, y, z, w;
    Vector3f xyz() const;
    Vector3f to_vector3() const; // divide by w
};

Design Rationale

Vector2f, Vector3f, and Vector4f are gryce-engine's custom vector types, providing full operator overloading and common geometric operations. They do not use SIMD instructions, keeping the code simple and portable. Vector3f's cross product follows the right-hand rule. All vector types support implicit conversion to float*, so they can be passed directly to OpenGL's glUniform*fv and Vulkan's push constants.

Mathematical Conventions

All angle operations (such as rotate_x, rotate_y, rotate_z) use degrees instead of radians. This differs from other graphics libraries (such as glm, which uses radians) and is done to stay consistent with common 3D editing tools (like Blender). Length and distance calculations use the standard Euclidean norm. normalize returns a zero vector when the length is 0, without throwing an exception.

Caveats

  • Vector types do not use SIMD, so they may be slower than glm for heavy computation.
  • cross uses the right-hand rule in a right-handed coordinate system.
  • lerp uses linear interpolation; slerp uses spherical interpolation and requires the input vectors to be normalized.
  • Vector4f's to_vector3 performs homogeneous division (divides by w) and returns a zero vector when w is 0.

gryce-engine

A very early-stage C++23 game framework. No editor. Not built from scratch — depends on GLFW, GLEW, Vulkan SDK, miniaudio, Jolt, Box2D, Assimp, and others. Handwritten parts: ECS, RHI, math, scene.
