PhysicalMaterial

PhysicalMaterial

Surface properties for physics bodies: friction, restitution, and density. Shared across multiple bodies.

#include <GryceRC/GrycePhysicalMaterial.h>

struct PhysicalMaterial {
    float friction = 0.3f;
    float restitution = 0.0f; // bounciness, 0 = no bounce, 1 = full bounce
    float density = 1.0f;

    static PhysicalMaterial ice()     { return {0.02f, 0.0f, 0.9f}; }
    static PhysicalMaterial rubber()  { return {0.8f, 0.9f, 1.1f}; }
    static PhysicalMaterial wood()    { return {0.4f, 0.2f, 0.6f}; }
    static PhysicalMaterial metal()   { return {0.2f, 0.1f, 7.8f}; }
    static PhysicalMaterial default_material() { return {}; }
};

// Usage in body creation
BodyDesc desc;
desc.material = PhysicalMaterial::rubber();
desc.mass = desc.material.density * volume;

Design Rationale

PhysicalMaterial is the physics material definition in gryce-engine, describing the friction, restitution, and density of an object's surface. It is not a component but rather a parameter used when creating a physics Body. PhysicalMaterial provides default values based on common materials (ice, rubber, wood, metal), but developers are free to define their own. In Jolt and Box2D, PhysicalMaterial's fields map to the corresponding backend's material properties. BuiltInPhysicsWorld2D uses a simplified material model that only supports friction and restitution.

Material Parameters

friction controls sliding resistance, ranging from 0.0 (frictionless) to 1.0+ (high friction). restitution controls bounciness: 0.0 means a perfectly inelastic collision, and 1.0 means a perfectly elastic collision. density is multiplied by volume to obtain mass, used for mass calculation of Dynamic rigid bodies. Static and Kinematic rigid bodies have their mass specified directly by the caller, ignoring density.

Caveats

  • friction can exceed 1.0, but some physics backends (such as Box2D) have undefined behavior when friction > 1.0.
  • restitution > 1.0 causes energy gain, which may lead to simulation instability.
  • density is measured in kg/m^3, consistent with real-world units (water = 1000).
  • PhysicalMaterial can be shared; multiple Bodies can use the same PhysicalMaterial instance.
