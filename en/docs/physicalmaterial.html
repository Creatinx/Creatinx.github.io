PhysicalMaterial

PhysicalMaterial

Surface properties for physics bodies: friction, restitution, and density. Shared across multiple bodies.

#include <GryceRC/GrycePhysicalMaterial.h>

struct PhysicalMaterial {
    float friction = 0.3f;
    float restitution = 0.0f; // bounciness, 0 = no bounce, 1 = full bounce
    float density = 1.0f;

    static PhysicalMaterial ice()     { return {0.02f, 0.0f, 0.9f}; }
    static PhysicalMaterial rubber()  { return {0.8f, 0.9f, 1.1f}; }
    static PhysicalMaterial wood()    { return {0.4f, 0.2f, 0.6f}; }
    static PhysicalMaterial metal()   { return {0.2f, 0.1f, 7.8f}; }
    static PhysicalMaterial default_material() { return {}; }
};

// Usage in body creation
BodyDesc desc;
desc.material = PhysicalMaterial::rubber();
desc.mass = desc.material.density * volume;

设计原理

PhysicalMaterial 是 gryce-engine 中的物理材质定义，描述物体表面的摩擦、弹性和密度。它不是组件，而是物理 Body 创建时的参数。PhysicalMaterial 的默认值基于常见材质（冰、橡胶、木头、金属），但开发者可以自由定义。在 Jolt 和 Box2D 中，PhysicalMaterial 的字段映射到对应后端的材质属性。BuiltInPhysicsWorld2D 使用简化版的材质模型，只支持 friction 和 restitution。

材质参数

friction 控制滑动阻力，范围 0.0（无摩擦）到 1.0+（高摩擦）。restitution 控制弹性，0.0 表示完全非弹性碰撞，1.0 表示完全弹性碰撞。density 与体积相乘得到质量，用于 Dynamic 刚体的质量计算。Static 和 Kinematic 刚体的 mass 由调用者直接指定，忽略 density。

注意事项

  • friction 可以大于 1.0，但某些物理后端（如 Box2D）在 friction > 1.0 时行为未定义。
  • restitution > 1.0 会导致能量增加，可能导致模拟不稳定。
  • density 的单位是 kg/m^3，与真实世界一致（水 = 1000）。
  • PhysicalMaterial 可以共享，多个 Body 可以使用同一个 PhysicalMaterial 实例。
