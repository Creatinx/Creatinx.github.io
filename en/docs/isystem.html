ISystem

ISystem

Base interface for all game logic systems. Override the lifecycle hooks and return a priority value. Higher priority = earlier execution within the same phase.

#include <GryceRC/GryceSystem.h>

class ISystem {
public:
    virtual ~ISystem() = default;

    virtual int priority() const { return 0; } // higher = earlier

    virtual void init() {}
    virtual void fixed_update(float dt) {}
    virtual void update(float dt) {}
    virtual void render(IRenderBackend* backend) {}
    virtual void cleanup() {}

    void set_world(World* world);
    World* world() const;
};

// Example custom system
class SpinSystem : public ISystem {
public:
    int priority() const override { return 100; }

    void update(float dt) override {
        for (auto [entity, transform] : world()->scene().query<Transform>()) {
            transform.rotation.y += 90.0f * dt;
        }
    }
};

Query API

Systems iterate over entities matching component filters via the query API.

// Query all entities with Transform and MeshRenderer
for (auto [entity, transform, mesh] : scene.query<Transform, MeshRenderer>()) {
    mesh.material.set_mat4("u_model", transform.matrix());
}

设计原理

ISystem 是 gryce-engine 中所有游戏逻辑系统的基接口。它定义了五个生命周期钩子：init、fixed_update、update、render、cleanup。World 在每个阶段按 priority() 降序调用所有系统的对应钩子。priority() 默认返回 0，值越大执行越早。这种设计允许系统之间的依赖管理，例如物理系统应该在游戏逻辑之前执行，渲染系统应该在所有逻辑之后执行。

生命周期

init 在 World::init() 时调用一次，用于系统初始化。fixed_update 以固定时间步长调用，适合物理模拟和确定性逻辑。update 以实际帧时间调用，适合动画和游戏逻辑。render 在每帧结束时调用，接收 IRenderBackend* 用于绘制。cleanup 在 World::cleanup() 时调用，用于释放资源。

注意事项

  • 不要在 init 中调用其他系统的接口，因为系统初始化顺序不确定。
  • fixed_update 的时间步长是固定的，不要假设它等于 delta_time。
  • render 方法中不要修改游戏状态，这可能导致不一致。
  • 系统的析构函数在 World 析构时自动调用，不需要手动清理。

gryce-engine

A very early-stage C++23 game framework. No editor. Not built from scratch — depends on GLFW, GLEW, Vulkan SDK, miniaudio, Jolt, Box2D, Assimp, and others. Handwritten parts: ECS, RHI, math, scene.
