AsyncLoader

Background-thread asset loader with progress callbacks and dependency tracking. GPU-safe upload via render thread pause.

#include <GryceRC/GryceAsyncLoader.h>

class AsyncLoader {
public:
    void init(size_t thread_count = 2);
    void shutdown();

    template<typename T>
    LoadTicket<T> load_async(const std::string& path);

    template<typename T>
    LoadTicket<T> load_async(const std::string& path, std::function<void(Handle<T>)> on_complete);

    void pause_render_thread(); // before GPU upload
    void resume_render_thread();

    size_t pending_count() const;
    size_t completed_count() const;
    float progress() const; // 0.0 - 1.0

    void wait_all();
    void cancel_all();
};

// LoadTicket usage
auto ticket = loader.load_async<Mesh>("res:/models/hero.fbx");
if (ticket.is_ready()) {
    auto mesh = ticket.get();
}

设计原理

AsyncLoader 是 gryce-engine 的异步资源加载器，使用 std::thread 池在后台线程加载资源。它支持两种加载模式：阻塞式（load_async 返回 LoadTicket，通过 is_ready() 检查）和回调式（load_async 传入 on_complete 回调）。AsyncLoader 内部使用生产者-消费者队列：主线程提交加载任务，后台线程从队列取出任务并执行。加载完成后，回调在主线程调用（通过轮询机制）。

GPU 上传同步

对于需要 GPU 上传的资源（如 Texture、Mesh），AsyncLoader 在后台线程读取文件后，调用 pause_render_thread() 暂停渲染线程，然后在主线程执行 GPU 上传，最后调用 resume_render_thread() 恢复。这种设计避免了多线程同时访问 GPU 资源，但会引入短暂的帧率下降。对于不频繁加载的场景，这种影响可以忽略。

注意事项

  • AsyncLoader 的线程数默认为 2，可通过 init() 参数调整。
  • LoadTicket 在资源加载完成前持有对任务的引用，不要丢弃未完成的 ticket。
  • cancel_all 会取消所有未开始的任务，正在执行的任务会继续到完成。
  • progress() 返回的是已提交任务的进度，不包括回调执行时间。
