GLMesh

GLMesh

Manages VAO, VBO, and EBO. Uploads vertex data and draws via glDrawArrays or indexed drawing.

#include <GryceRC/GryceGLMesh.h>

class GLMesh {
public:
    void upload_vertices(const void* data, size_t size, const VertexLayout& layout);
    void upload_indices(const void* data, size_t size, IndexType type);

    void set_layout(const VertexLayout& layout);
    void bind();
    void unbind();

    void draw();
    void draw_indexed(uint32_t count);

    uint32_t vao() const;
    uint32_t vbo() const;
    uint32_t ebo() const;
    uint32_t index_count() const;
};

Example

float vertices[] = {
    -0.5f, -0.5f, 0.0f,
     0.5f, -0.5f, 0.0f,
     0.0f,  0.5f, 0.0f,
};
uint32_t indices[] = { 0, 1, 2 };

VertexLayout layout;
layout.stride = sizeof(float) * 3;
layout.attributes.push_back({"a_pos", 0, 3, AttributeType::Float, false, 0});

GLMesh mesh;
mesh.upload_vertices(vertices, sizeof(vertices), layout);
mesh.upload_indices(indices, sizeof(indices), IndexType::Uint32);
mesh.draw_indexed(3);

设计原理

GLMesh 管理 OpenGL VAO、VBO 和 EBO 的生命周期。它在 upload_vertices() 中创建并填充 VBO，在 upload_indices() 中创建并填充 EBO。VertexLayout 描述了顶点属性的格式和偏移，upload_vertices 使用这个信息在 VAO 上配置顶点属性指针。这种设计允许同一 GLMesh 对象在不同 shader 中复用，只要 vertex layout 兼容。

顶点布局

VertexLayout 由 stride（顶点大小，字节）和 attributes（属性数组）组成。每个属性包含 location、components、type、normalized 和 offset。upload_vertices 根据这些信息调用 glVertexAttribPointer 配置 VAO。如果后续 shader 的 attribute location 与 VertexLayout 中定义的不匹配，渲染结果将不正确。

注意事项

  • VAO 在 upload_vertices 时创建，如果 layout 改变需要重新调用 upload_vertices。
  • draw_indexed 的 count 参数是索引数量，不是顶点数量。
  • IndexType 必须是 Uint16 或 Uint32，Uint8 不受支持。
  • GLMesh 析构时自动删除 VAO、VBO 和 EBO。
