JoltPhysicsWorld3D

JoltPhysicsWorld3D

3D rigid body physics using Jolt Physics 5.2. Supports static, dynamic, and kinematic bodies. Raycast and collision detection included.

#include <GryceRC/GryceJoltPhysics.h>

class JoltPhysicsWorld3D : public IPhysicsWorld3D {
public:
    bool init() override;
    void shutdown() override;

    void step(float dt) override;
    void set_gravity(const Vector3f& gravity) override;

    BodyHandle create_body(const BodyDesc& desc) override;
    void destroy_body(BodyHandle handle) override;

    void set_position(BodyHandle handle, const Vector3f& pos) override;
    void set_rotation(BodyHandle handle, const Quaternion& rot) override;
    void set_velocity(BodyHandle handle, const Vector3f& vel) override;

    Vector3f get_position(BodyHandle handle) const override;
    Vector3f get_velocity(BodyHandle handle) const override;

    void add_force(BodyHandle handle, const Vector3f& force) override;
    void add_impulse(BodyHandle handle, const Vector3f& impulse) override;

    RaycastResult raycast(const Vector3f& from, const Vector3f& to) const override;
    std::vector<CollisionResult> check_collision(BodyHandle handle) const override;
};

struct BodyDesc {
    BodyType type = BodyType::Dynamic; // Static, Dynamic, Kinematic
    ShapeHandle shape;
    Vector3f position;
    float mass = 1.0f;
    PhysicalMaterial material;
};

Design Rationale

JoltPhysicsWorld3D is gryce-engine's 3D physics backend, based on Jolt Physics 5.2. It implements the IPhysicsWorld3D interface, providing rigid body dynamics, collision detection, raycasting, and a constraint system. JoltPhysicsWorld3D creates Jolt's PhysicsSystem, BodyInterface, and ContactListener in init(). In step(), it calls PhysicsSystem::Update to advance the physics simulation at a fixed time step. Jolt's job system uses std::thread for multi-threaded physics simulation, utilizing all available CPU cores by default.

Body Types

Jolt supports three rigid body types: Static (immovable, infinite mass), Dynamic (affected by forces and collisions), and Kinematic (position controlled by code, not affected by forces but affects other bodies). These are specified via the type field in BodyDesc. PhysicalMaterial's friction and restitution map to the body's friction and restitution_scale in Jolt.

Caveats

  • Jolt's step() must be called at a fixed time step; otherwise, the simulation becomes unstable.
  • raycast returns a RaycastResult containing the hit body, hit point, and hit normal.
  • add_force applies a force in world-space coordinates; add_impulse applies an impulse (instantaneous force).
  • Shapes in Jolt are referenced via ShapeHandle and are managed internally by JoltPhysicsWorld3D.
