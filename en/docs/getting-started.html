Getting Started

Prerequisites

Before building Gryce Engine, ensure your development environment meets the following requirements. The framework is developed and tested on Windows with MSYS2 UCRT64. GCC 13 or later with full C++23 support is required because the codebase uses features like concepts, ranges, and designated initializers extensively. CMake 3.28+ is needed for the modern preset and module support. Ninja is the recommended generator for fast parallel builds.

  • Windows 10/11 with MSYS2 UCRT64 environment
  • GCC 13+ with C++23 support
  • CMake 3.28+
  • Ninja build system
  • Python 3.10+ for the build script

Installation

Clone the repository and install dependencies via MSYS2, then run the build script. The build script auto-detects GCC location, configures CMake with the correct toolchain, and runs Ninja with parallel jobs. After the build completes, the editor executable and engine library are available in the build directory.

git clone https://github.com/Creatinx/Gryce-Engine.git
cd Gryce-Engine
pacman -S mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja mingw-w64-ucrt-x86_64-glew mingw-w64-ucrt-x86_64-glfw
python build.py --jobs 8
./build/Debug/bin/Debug/gryce.exe

Project Structure

The repository is organized into several top-level directories. The core/ directory contains the engine library (GryceRC) with all handwritten systems: ECS, RHI, math, scene, audio, physics integration, and asset pipeline. The editor/ directory is currently a placeholder executable. The tests/ directory contains unit tests for the math library and ECS. The examples/ directory contains sample projects demonstrating rendering and physics.

Gryce-Engine/
├── core/          # Engine library (GryceRC)
├── editor/        # Editor executable (placeholder)
├── tests/         # Unit tests
├── examples/      # Sample projects
├── cmake/         # CMake modules
├── docs/          # Documentation
└── build.py       # One-click build script

Build Modes

The build script supports multiple modes: Debug (default), Release, and Clean. Debug builds include full assertions, address sanitizer, and debug symbols. Release builds enable -O3, LTO, and strip debug symbols. The --tests flag enables the GoogleTest-based unit test suite.

Overview

gryce-engine is a very early-stage C++23 game framework. There is no editor. It is not built from scratch — it depends on GLFW, GLEW, Vulkan SDK, miniaudio, Jolt, Box2D, Assimp, and others. The handwritten parts are ECS, RHI, math, and scene layers.

Prerequisites

  • Windows 10/11 with MSYS2 UCRT64 environment
  • GCC 13+ with C++23 support
  • CMake 3.28+
  • Ninja build system
  • Python 3.10+

Installation

python build.py --jobs 8
./build/Debug/bin/Debug/gryce.exe

Minimal Example

#include <GryceRC/GryceEngine.h>
using namespace gryce_engine;

int main() {
    Window window("Gryce Demo", 1280, 720);
    auto context = RenderContext::create(RenderBackend::OpenGL, window);
    World world;
    auto camera = world.scene().create_entity("Camera");
    camera.add_component<Camera>(60.0f, 16.0f/9.0f, 0.1f, 1000.0f);
    auto mesh = AssetManager::load<Mesh>("res:/models/cube.obj");
    while (window.is_open()) {
        window.poll_events();
        world.update(window.delta_time());
        world.render();
        context->present();
    }
    return 0;
}

设计原理

gryce-engine 采用手写核心层 + 成熟第三方库的策略。ECS、RHI、数学库和场景层是手写实现，保证了对底层行为的完全控制。窗口/输入（GLFW）、OpenGL 加载（GLEW）、音频（miniaudio）、物理（Jolt/Box2D）、模型加载（Assimp）使用第三方库，避免重复造轮子。这种设计在开发效率和可控性之间取得了平衡。

第三方依赖

必需依赖：GLFW（窗口和输入）、GLEW（OpenGL 扩展加载）、CMake、Ninja、GCC 13+。可选依赖：Vulkan SDK（Vulkan 后端）、VMA（Vulkan 内存管理）、Jolt Physics（3D 物理）、Box2D（2D 物理）、Assimp（模型加载）、miniaudio（音频）、stb_image（纹理加载）、nlohmann/json（场景序列化）、glm（可选数学库）、spdlog（日志）。

注意事项

  • 首次构建需要安装 MSYS2 和基本开发工具，耗时约 10-15 分钟。
  • 如果只需要 OpenGL 后端，可以跳过 Vulkan SDK 安装。
  • build.py 会自动将 MSYS2 的 DLL 复制到输出目录，确保可执行文件能直接运行。
  • Debug 构建包含断言和地址检查，适合开发；Release 构建适合发布。
