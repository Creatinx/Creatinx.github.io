Entity

Entity

Lightweight handle (uint32_t ID + generation) representing an object in the scene. Supports parent-child hierarchy and component attachment. Null entity has ID 0.

#include <GryceRC/GryceEntity.h>

class Entity {
public:
    Entity() = default; // null entity
    Entity(EntityID id, uint32_t generation);

    bool is_valid() const;
    EntityID id() const;
    uint32_t generation() const;

    Entity create_child(const std::string& name);
    void set_parent(Entity parent);
    Entity parent() const;
    std::vector<Entity> children() const;

    template<typename T, typename... Args>
    T& add_component(Args&&... args);

    template<typename T>
    T& get_component();

    template<typename T>
    bool has_component() const;

    template<typename T>
    void remove_component();

    void destroy(); // recursively destroys children
    std::string name() const;
    void set_name(const std::string& name);

    bool operator==(const Entity& other) const;
    bool operator!=(const Entity& other) const;
};

Example

auto player = world.create_entity("Player");
player.add_component<Transform>(Vector3f(0,0,0));
player.add_component<MeshRenderer>(mesh_handle, shader_handle);
player.add_component<RigidBody>(1.0f, true); // mass, dynamic

auto gun = player.create_child("Gun");
gun.add_component<Transform>(Vector3f(0.5f, 0, 0));

设计原理

Entity 是一个轻量级的句柄（uint32_t ID + generation），而不是指向实际对象的指针。这种设计允许 Entity 在组件被销毁后仍然保持有效检查（通过 generation 比较）。Entity 支持父子层级，子实体的 Transform 会自动继承父实体的变换。每个 Entity 可以附加多个组件，组件类型通过模板参数指定，类型安全且在编译期解析。

组件操作

add_component 使用完美转发将参数传递给组件构造函数。get_component 返回组件的引用，如果组件不存在则抛出异常。has_component 在 O(1) 时间内检查组件是否存在。remove_component 立即销毁组件并释放其存储空间。所有组件操作都通过 ComponentStore 的稀疏集实现，保证常数时间复杂度。

注意事项

  • Entity 默认构造为空句柄（id = 0, generation = 0）。
  • set_parent 会触发子树的世界矩阵重新计算。
  • destroy() 递归销毁所有子实体，谨慎使用。
  • 不要在组件的构造函数中调用其他组件的方法。
