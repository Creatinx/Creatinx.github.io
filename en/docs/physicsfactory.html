PhysicsFactory

PhysicsFactory

Factory for creating physics worlds. Selects backend at runtime based on CMake flags and availability.

#include <GryceRC/GrycePhysics.h>

class PhysicsFactory {
public:
    static std::unique_ptr<IPhysicsWorld2D> create_world_2d(PhysicsBackend backend);
    static std::unique_ptr<IPhysicsWorld3D> create_world_3d(PhysicsBackend backend);

    static bool is_backend_available(PhysicsBackend backend);
    static std::vector<PhysicsBackend> available_backends();
};

enum class PhysicsBackend {
    Jolt3D,      // 3D rigid body
    Box2D,       // 2D physics
    BuiltIn2D,   // lightweight custom 2D
    None
};

Example

auto world3d = PhysicsFactory::create_world_3d(PhysicsBackend::Jolt3D);
auto world2d = PhysicsFactory::create_world_2d(PhysicsBackend::Box2D);

if (!PhysicsFactory::is_backend_available(PhysicsBackend::Jolt3D)) {
    world3d = PhysicsFactory::create_world_3d(PhysicsBackend::None);
}

设计原理

PhysicsFactory 是 gryce-engine 中物理后端的工厂类，负责根据 CMake 配置和运行时可用性创建物理世界实例。它支持三种后端：Jolt Physics 3D、Box2D 2D 和 BuiltIn 2D。在运行时，PhysicsFactory 通过 is_backend_available() 检查后端是否编译进引擎（通过 CMake 选项控制）。这种设计允许同一套游戏代码在不同构建配置下运行，无需修改物理相关逻辑。

后端选择策略

当调用 create_world_2d 或 create_world_3d 时，PhysicsFactory 首先检查请求的后端是否可用。如果不可用，返回 nullptr 或降级到 None 后端。None 后端是一个空实现，所有方法均为空操作，允许游戏在没有物理库的情况下编译和运行。available_backends() 返回当前构建中可用的后端列表，用于 UI 中的后端选择。

注意事项

  • PhysicsFactory 不是线程安全的，所有调用必须在主线程执行。
  • is_backend_available 只检查编译时可用性，不检查运行时库是否加载成功。
  • create_world_3d 目前只支持 Jolt3D 和 None 两种后端。
  • 物理世界的生命周期由调用者管理，PhysicsFactory 只负责创建。
